Class Hierarchy

  • hid.HIDDevice: absract class representing a HID device on the host computer
    • hid.osx.OSXHIDDevice: class representing a HID device on the host (OS X) computer
  • ioLabs.Commands: class to handle sending reports to device and parsing incoming reports.
  • ioLabs.dict_struct: simple class that takes keyword arguments and uses them to create fields on itself
  • ioLabs.messages: class to handle message id lookup, and packing message objects into binary
  • object: The most base type
    • ioLabs.Interrupt: either int0 or int1 on the USBBox
      • ioLabs.VoiceKey: object representing the voice input on the USBBox
    • ioLabs.Serial: serial port on the USBBox
    • ioLabs.Port0_2: class representing ports 0 and 2 (leds)
    • ioLabs.USBBox: the USBBox itself
    • ioLabs.Buttons: class that represents the buttons on the USBBox
    • ioLabs.Line: base class for Lines (individual parts of ports)
